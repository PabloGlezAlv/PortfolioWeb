RTBEngine - Game Engine Architecture

  • Category: Tools / Game Engine
  • Tech Stack: C++17, OpenGL 4.3+, SDL2, Bullet Physics, FMOD, Lua, ImGui
  • Platform: Windows x64

Executive Summary

RTBEngine is a custom 3D game engine designed for Windows, built with C++ and featuring an Entity Component System (ECS) architecture. The engine integrates industry-standard libraries for graphics, physics, and audio, while maintaining custom implementations for core systems like mathematics and ECS.

Important: RTBEngine is a standalone engine library. Games will be developed as separate projects that link against this engine.

System Architecture

The engine uses a GameObject-Component architecture where scenes are defined in .lua files using a declarative syntax.

Entity Component System (ECS)

Scene
  └── GameObject[]
        ├── Transform (built-in)
        └── Component[]
              ├── MeshRenderer
              ├── RigidBodyComponent
              ├── LightComponent
              └── ...

Scripting & Serialization

Scenes are loaded from Lua files. The SceneLoader instantiates components via a ComponentRegistry.

function CreateScene()
    return {
        name = "MyScene",
        gameObjects = {
            {
                name = "Player",
                components = {
                    { type = "MeshRenderer", model = "player.fbx" },
                    { type = "RigidBodyComponent", mass = 1.0 }
                }
            }
        }
    }
end

Editor Architecture

RTBEngine Editor is a visual editor featuring:

  • Dockable Panel Layout with ImGui
  • Scene Hierarchy and Inspector panels
  • 3D Scene View with gizmos
  • C++ Hot Reload via DLL plugin system

The editor runs the game engine in a controlled environment, rendering the scene to framebuffers that are displayed within ImGui panels.
